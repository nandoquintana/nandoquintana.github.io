Introducción a Zope

  • ¿Qué es Zope?
    • Plataforma para crear aplicaciones web
      • (interfaz de usuario = páginas web)
    • Está escrito en Python
    • Publicado bajo una licencia libre:
      • Zope Public License (http://www.zope.org/Resources/License/)
  • http://www.zope.org
    •                                    

Introducción a Zope

  • Z Object Publishing Environment =
    • programación orientada a objetos (Python)
    • + persistencia de objetos
    • + acceso remoto a los objetos (http, ftp, xml-rpc, etc.)
  • Se le suele achacar una larga curva de aprendizaje

Introducción a Zope

  • Incluye:
    • interfaz de administración web (ZMI)
    • una base de datos OO transaccional
    • un servidor (web, ftp, xml-rpc, webdav)
  • Permite:
    • fácil recuperación frente a errores
    • acceso a los recursos de forma concurrente

Introducción a Zope

  • En 1996, Jim Fulton de Digital Creations ideó el nucleo de Zope
  • Se desarrolla una aplicación comercial llamada 'Principia'
  • En 1998 se libera su código bajo el nombre de 'Zope'
    • 1999 - Simon Michel - Zwiki
    • 2002 - Alexander Limi, Alan Runyan y Vidar Andersen - Plone
    • 2003 - Atsushi Shibata - CoreBlog

Introducción a Zope

  • Zope está diseñado para delegar, de forma segura, el control de un sitio web a diferentes grupos de usuarios
  • Se puede administrar de forma remota utilizando un navegador web.
    • se parece mucho a un administrador de ficheros (ej:nautilus, Hasefroch explorer)

Introducción a Zope

  • Un sitio web Zope está compuesto por objetos (Z objects) en lugar de ficheros de texto (ej:php)
  • Los objetos más básicos que ofrece Zope son:
    • carpetas, documentos, imagenes,
      plantillas, etc.
  • Existen multitud de productos que permiten utilizar nuevos tipos de objetos, por ejemplo:
    • un wiki (Zwiki), un web log (CoreBlog),
      un portal (Plone), etc.
